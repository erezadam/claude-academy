# Publish — כתיבת מאמר מתוך וורדיקט טריאז' מאושר

אתה כותב או מעדכן **מאמר אחד** ב-`knowledge-base/` על בסיס וורדיקט שהופק בסקיל `triage` **ואושר במפורש על-ידי המשתמש**. לסקיל הזה תופעת-לוואי (כתיבת קבצים + פתיחת PR) ולכן הוא מופעל ידנית בלבד — לעולם לא אוטונומית, ולעולם לא בלי וורדיקט מאושר.

הכללים המחייבים מוגדרים ב-`.claude/academy-update-spec.md` — **קרא אותו לפני הכתיבה ופעל לפיו; אל תמציא כללים.**

## קלט נדרש

וורדיקט מאושר הכולל: **create או update** (ואם update — הקובץ הקיים), **קטגוריה** (לפי מיפוי שלב 4 של ה-spec), **source_url רשמי** (code.claude.com/docs בלבד), ו-**layer**. חסר אחד מהם — עצור ובקש השלמה; אל תנחש.

## תהליך

### 1. משיכת המקור
משוך את ה-source_url (curl / WebFetch). המקור הוא הבסיס הבלעדי לתוכן — **source-fidelity: רק מה שמופיע במקור נכנס למאמר.** אין מקור זמין → עצור, אל תכתוב.

### 2. כתיבה לפי ה-spec
- **create:** צור `knowledge-base/<category>/<slug>.md` עם ה-frontmatter המלא משלב 5 של ה-spec: `title` (עברית), `category`, `layer`, `last_verified` (תאריך היום), `status: needs-review`, `source_url`, `related` (מלא כשיש קשרים ברורים).
- **update:** ערוך את הקובץ הקיים שצוין בוורדיקט — אל תיצור קובץ כפול. עדכן `last_verified` להיום ו-`status: needs-review`.
- **מבנה גוף** לפי ה-spec: פסקת פתיחה, `## מה זה עושה / למה זה שימושי`, `## איך משתמשים` עם לפחות בלוק קוד אחד אמיתי מהמקור, וסעיפים נוספים רק אם המקור מספק.
- **אילוצי parseMarkdown:** h2, h3, פסקאות, code inline/block, טבלה, hr בלבד. אסור: רשימות מקוננות, blockquote, תמונות, h1/h4. תוכן שדורש מבנה לא נתמך — נסח מחדש במבנה נתמך; אם בלתי-אפשרי, עצור וסמן "דורש עיצוב ידני".
- **קטגוריה:** רק מהקטגוריות הקיימות לפי הוורדיקט. אין ליצור קטגוריה חדשה.

### 3. אימות עצמי (שלב 6.5 של ה-spec — חובה)
לפני פתיחת PR: עבור על **כל** פקודה, דגל, מפתח-הגדרה וטענה במאמר, וודא שכל אחד מופיע במפורש ב-source_url (או בעמוד רשמי אחר תחת code.claude.com/docs). טענה שלא אומתה — הזיה: הסר או תקן. בנוסף הרץ `npm run build` וודא שהוא ירוק; build שבור — תקן לפני PR.

### 4. ענף + PR
- ענף תוכן חדש: `academy-content/<slug>` (מ-main עדכני). commit + push.
- פתח PR עם דוח: מה נוצר/עודכן, קטגוריה, layer, קישור למקור, ותוצאת אימות שלב 6.5.
- **עצור ודווח את קישור ה-PR.** אל תמזג, אל תיגע ב-main.

## איסורים מוחלטים

- לעולם לא `source_url` בדומיין צד-שלישי — רק code.claude.com/docs.
- אין לכתוב ל-main ואין למזג — הפלט היחיד הוא PR פתוח הממתין לאישור.
- מאמר אחד לכל הפעלה; אין לגעת ב-pipeline, ב-`data/` (מלבד אם ה-spec מחייב changelog לפריט), או בקבצים שאינם המאמר שבוורדיקט.
