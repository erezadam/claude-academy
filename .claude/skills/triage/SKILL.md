# Triage — בחינת פריט נכנס לאקדמיה

אתה מבצע **בחינה בלבד** של פריט מועמד (URL / טקסט / קובץ) עבור knowledge-base של claude-academy. הכללים המחייבים — allowlist מקורות, סכמת frontmatter, קטגוריות, אילוצי markdown — מוגדרים ב-`.claude/academy-update-spec.md`. **קרא אותו קודם ופעל לפיו; אל תמציא כללים מחדש.**

## איסורים מוחלטים

- אין לכתוב או לערוך שום קובץ תחת `knowledge-base/` או `data/`.
- אין לפתוח PR של תוכן ואין לגעת ב-main.
- `source_url` מוצע חייב להיות עמוד רשמי תחת code.claude.com/docs (או docs.claude.com / anthropic.com הרשמיים). **לעולם לא דומיין צד-שלישי** — בלוג, פוסט, סרטון וכד' יכולים לשמש רק כ-inspiration, לא כמקור.

## תהליך

### 1. סיווג ובדיקת סקופ
זהה את סוג הקלט (URL / טקסט חופשי / קובץ) ואת הנושא. הסקופ המותר לטריאז' זה: **Claude Code ו-Git בלבד**. נושא מחוץ לסקופ → עצור והחזר דחייה מנומקת (מה הנושא, למה מחוץ לסקופ). שים לב: לפי ה-spec, סוכן העדכון השבועי לא נוגע בקטגוריית git — אם הפריט הוא git, ציין זאת כהסתייגות בוורדיקט.

### 2. ציד מקור רשמי
חפש את העמוד הרשמי המתועד ב-allowlist (התחל מ-`https://code.claude.com/docs/llms.txt` ומהעמודים תחת `code.claude.com/docs/en/...`). משוך את העמוד (curl / WebFetch) וחלץ **את המשפט הרשמי המדויק** שתומך בכל טענה מרכזית בפריט. אין עמוד רשמי → אין מאמר (כלל הברזל של ה-spec).

### 3. שתי שאלות נפרדות
ענה על כל אחת בנפרד, עם ציטוטים:
- **(א) האם המושג מתועד רשמית?** — קיים עמוד רשמי שמכסה את היכולת עצמה?
- **(ב) האם הניסוח, המונחים והטענות הספציפיות בפריט תואמים למקור?** — כל מונח, שם-פיצ'ר, דגל או טענה שלא מופיעים מילולית בעמוד רשמי → סמן במפורש: **"ניסוח צד-שלישי — inspiration בלבד"**. אל תיתן למונח מומצא לחלחל להצעת המאמר.

### 4. בדיקת כפילות
בנה אינדקס dedup כמו בשלב 2 של ה-spec: עבור על `knowledge-base/**/*.md`, חלץ `source_url` מנורמל (בלי `.md`, בלי `/en/`, בלי סלאש סופי). אם המקור הרשמי כבר מכוסה — **עדכון (update) עדיף על יצירה (create)**; ציין את הקובץ הקיים.

### 5. וורדיקט מובנה
החזר טבלה/רשימה עם:
- **שווה מאמר?** כן / לא / גבולי + נימוק.
- **create או update** (+ קובץ קיים אם update).
- **קטגוריה מוצעת** — לפי סדר המיפוי בשלב 4 של ה-spec (scheduling → workflows → project-docs → guides → claude-code). אין להמציא קטגוריה חדשה — רק להמליץ.
- **layer מוצע** ו-**source_url מוצע** (רשמי בלבד).
- **דגלי parseMarkdown** — האם התוכן ידרוש מבנה לא נתמך (רשימות מקוננות, blockquote, תמונות, h1/h4)? אם כן — "דורש עיצוב ידני".
- **תלות-גרסה** — האם הטענות תלויות בגרסת Claude Code מסוימת / פיצ'ר בשלב preview?
- **רשימת "ניסוח צד-שלישי"** — כל מונח/טענה שלא אומתו מילולית במקור.

### 6. עצירה
הצג את הוורדיקט ו**עצור**. אל תכתוב מאמר, אל תפתח PR של תוכן. כתיבה בפועל מתבצעת רק לאחר אישור אנושי מפורש, ואז לפי ה-spec (כולל שלב 6.5 — אימות תוכן מול המקור).
