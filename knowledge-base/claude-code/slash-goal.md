הפקודה `/goal` מגדירה תנאי סיום, ו-Claude ממשיך לעבוד בצורה עצמאית עד שהתנאי מתקיים — בלי שתצטרכו להפעיל כל תור ידנית.

## מה זה עושה / למה זה שימושי

`/goal` מאפשר ל-Claude להמשיך ולעבוד על משימה מרובת תורות עד שתנאי שהגדרתם מתקיים. אחרי כל תור, מודל קטן ומהיר בודק אם התנאי הושג — אם לא, Claude מתחיל תור נוסף במקום להחזיר שליטה אליכם. המטרה מתנקה אוטומטית ברגע שהתנאי מתקיים.

מתאים במיוחד למשימות גדולות עם מצב סיום ניתן לאימות:

- העברת מודול ל-API חדש עד שכל נקודות הקוד מקמפלות ובדיקות עוברות
- מימוש מסמך עיצוב עד שכל קריטריוני הקבלה מתקיימים
- ריקון תור issues עד שהתור ריק
- פיצול קובץ גדול לפי גודל מקסימלי

## איך משתמשים

### הגדרת מטרה

```text
/goal all tests in test/auth pass and the lint step is clean
```

הגדרת מטרה מתחילה תור מיידית, עם התנאי עצמו כהנחיה. בזמן שהמטרה פעילה, מופיע אינדיקטור `◎ /goal active` עם משך הזמן שרצה.

### כתיבת תנאי אפקטיבי

המעריך שופט את התנאי על בסיס מה ש-Claude הציג בשיחה בלבד — הוא לא מריץ פקודות ולא קורא קבצים בעצמו. כתבו תנאי שפלט Claude יכול להדגים:

- **מצב סיום מדיד אחד**: תוצאת בדיקה, קוד יציאה של build, ספירת קבצים, תור ריק
- **אימות מוצהר**: כיצד Claude אמור להוכיח זאת, לדוגמה `` `npm test` exits 0 ``
- **אילוצים חשובים**: מה לא צריך להשתנות בדרך

לגבול מספר תורות ניתן לכלול בתנאי: `or stop after 20 turns`.

### בדיקת מצב

```text
/goal
```

ללא ארגומנטים, מציג את התנאי, משך הריצה, מספר תורות, הוצאה בטוקנים, והסיבה האחרונה של המעריך.

### ביטול מטרה

```text
/goal clear
```

גם `stop`, `off`, `reset`, `none`, ו-`cancel` מתקבלים כ-aliases. הרצת `/clear` לפתיחת שיחה חדשה גם מנקה מטרה פעילה.

### חידוש עם מטרה פעילה

מטרה שהייתה פעילה בסיום הסשן משוחזרת בעת חידוש עם `--resume` או `--continue`. ספירת התורות, הטיימר, ובסיס הטוקנים מתאפסים.

### הרצה לא-אינטראקטיבית

```bash
claude -p "/goal CHANGELOG.md has an entry for every PR merged this week"
```

## השוואה לגישות אחרות

| גישה | התור הבא מתחיל כאשר | נעצר כאשר |
| :--- | :--- | :--- |
| `/goal` | התור הקודם הסתיים | מודל מאשר שהתנאי מתקיים |
| `/loop` | מרווח זמן חלף | אתם עוצרים, או Claude מחליט |
| Stop hook | התור הקודם הסתיים | הסקריפט או ה-prompt שלכם מחליט |

`/goal` ו-auto mode משלימים זה את זה: auto mode מסיר אישורי כלים בתוך כל תור, ו-`/goal` מסיר אישורי תורות.
